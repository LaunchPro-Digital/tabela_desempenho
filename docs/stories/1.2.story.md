## Notas Técnicas
- **Migrado para Supabase Cloud:** `pzehsvanabwgtuqlcppk.supabase.co`, schema `public`
- Originalmente self-hosted em `database.ordusdigital.com.br`, schema `desempenho`
- **Decisão: PostgreSQL function em vez de Edge Function** — Implementado como `public.manage_user()` com SECURITY DEFINER, que manipula diretamente `auth.users` e `auth.identities`. Chamado via `supabase.rpc('manage_user', params)`.
- **Auth users no Supabase Cloud:** NÃO podem ser criados via INSERT direto em `auth.users`. Devem ser criados via Dashboard ou Admin API (GoTrue). INSERTs manuais causam erro 500 no login.
- Hash de senha usa pgcrypto: `crypt(password, gen_salt('bf'))`
- O campo `email` na interface User serve para o login e display no AdminPanel
