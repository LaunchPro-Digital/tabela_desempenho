# Story 2.2 — Migrar manage_user CREATE para Edge Function

**Epic:** 2.0 — Hardening & Production-Ready
**Status:** Done
**Complexidade:** M (Medium)
**Agente:** @dev

## Objetivo
Substituir o INSERT direto em `auth.users` na action `create` do `manage_user` por uma Supabase Edge Function que usa a Admin API (GoTrue), garantindo compatibilidade com Supabase Cloud.

## Acceptance Criteria

### Edge Function
- [x] AC1: Edge Function `manage-user` criada em `supabase/functions/manage-user/`
- [x] AC2: Action `create` usa `supabase.auth.admin.createUser()` em vez de INSERT direto
- [x] AC3: Action `delete` usa `supabase.auth.admin.deleteUser()`
- [x] AC4: Action `reset_password` usa `supabase.auth.admin.updateUserById()`
- [x] AC5: Edge Function usa `SUPABASE_SERVICE_ROLE_KEY` server-side (nunca exposto ao client)

### Client-Side
- [x] AC6: `supabaseClient.ts` chama Edge Function via `supabase.functions.invoke('manage-user')`
- [x] AC7: AdminPanel funciona sem alterações na interface

### Migração
- [x] AC8: PostgreSQL function `manage_user` removida ou deprecated
- [x] AC9: Criação de usuário via AdminPanel funciona no Supabase Cloud (testado)

### Validação
- [x] AC10: Build sem erros
- [x] AC11: Testes passando

## Arquivos Impactados
- `supabase/functions/manage-user/index.ts` — criar Edge Function
- `supabaseClient.ts` — alterar `manageUser()` para chamar Edge Function
- `supabase/migration_cloud.sql` — deprecar manage_user function

## Notas Técnicas
- Supabase Cloud NÃO permite INSERT direto em `auth.users` — GoTrue gerencia essa tabela
- INSERTs diretos causam erro 500 no login (identities table não é populada)
- Edge Functions têm acesso nativo ao `SUPABASE_SERVICE_ROLE_KEY`
- A Admin API do GoTrue cria user + identity + password hash corretamente
- Deploy requer: `supabase functions deploy manage-user`

### Change Log
- `supabase/functions/manage-user/index.ts`: Edge Function com CORS, verificação JWT, check de role PARTNER, 3 handlers:
  - `create`: `auth.admin.createUser()` + INSERT em `app_users` + rollback se insert falhar
  - `delete`: lookup `auth_id` → `auth.admin.deleteUser()` + DELETE de `app_users`
  - `reset_password`: lookup `auth_id` → `auth.admin.updateUserById({ password: 'ordus2025' })` + `password_changed = false`
- `supabaseClient.ts`: Troca de `supabase.rpc('manage_user', params)` para `supabase.functions.invoke('manage-user', { body: params })`
- `supabase/migration_deprecate_manage_user.sql`: `DROP FUNCTION IF EXISTS public.manage_user(...)` com instruções de rollback
