# Story 1.4 — Tailwind Build-time

**Epic:** 1.0 — Tabela de Desempenho: Produção & Segurança
**Status:** Done
**Complexidade:** M (Medium)
**Agente:** @dev

## Objetivo
Migrar Tailwind CSS do CDN (runtime) para build-time com PostCSS/Vite plugin, preservando 100% do tema visual atual (cores brand, fonte Inter, dark mode, patterns).

## Acceptance Criteria

- [x] AC1: Dependências instaladas: `tailwindcss`, `@tailwindcss/vite`
- [x] AC2: `tailwind.config.ts` criado com tema brand completo:
  - Colors: purple (#9A11E9), purpleDark (#7F2DC1), green (#00A47B), greenDark (#006653), grey (#6A6A6A), black (#000000), offWhite (#F8FAFC), darkBg (#0f172a), darkCard (#1e293b), darkBorder (#334155)
  - Font: Inter (sans-serif)
  - Dark mode: `class`
- [x] AC3: Arquivo `src/index.css` criado com:
  - `@import "tailwindcss"`
  - Custom CSS: `.bg-brand-pattern` (light + dark), `.animate-fade-in` keyframe
  - Google Fonts Inter import
- [x] AC4: `index.tsx` importa `./index.css`
- [x] AC5: `index.html` limpo:
  - Removido: `<script src="https://cdn.tailwindcss.com">`
  - Removido: `<script>tailwind.config = {...}</script>`
  - Removido: `<style>` inline (movido para index.css)
- [x] AC6: Todas as classes Tailwind existentes continuam funcionando (visual idêntico)
- [x] AC7: Dark mode continua funcionando via toggle
- [x] AC8: Build produz CSS otimizado com purge (apenas classes usadas) — 39.04 kB gzip: 7.69 kB
- [x] AC9: `npm run build` sem erros — ✓ built in 2.01s
