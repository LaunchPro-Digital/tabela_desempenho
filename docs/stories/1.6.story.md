# Story 1.6 — Destaques Dinâmicos da Semana

**Epic:** 1.0 — Tabela de Desempenho: Produção & Segurança
**Status:** Done
**Complexidade:** S (Small)
**Agente:** @dev

## Objetivo
Substituir a seleção hardcoded dos "Destaques da Semana" no Dashboard por um algoritmo que identifica os membros com melhor performance real no trimestre.

## Acceptance Criteria

- [x] AC1: Função `getWeeklyHighlights(users, entries, topN)` criada em `services/calculator.ts`
- [x] AC2: Algoritmo de pontuação:
  - Para cada usuário, calcular status de cada métrica via `calculateMetricStatus()`
  - Pontuar: green=3, yellow=2, red=1, gray=0
  - Score total = soma dos pontos de todas as métricas
- [x] AC3: Ordenação por score descendente, top `topN` (default 2) retornados
- [x] AC4: Desempate: maior valor percentual da métrica primária (primeira métrica)
- [x] AC5: Dashboard.tsx usa `getWeeklyHighlights()` em vez de slice hardcoded
- [x] AC6: Cards de destaque exibem o score ou status visual correto
- [x] AC7: Funciona corretamente com 0 entries (nenhum destaque exibido)

## Arquivos Impactados
- `services/calculator.ts` — nova função `getWeeklyHighlights()`
- `components/Dashboard.tsx` — substituir lógica hardcoded de destaques

## Notas Técnicas
- Atualmente no Dashboard.tsx os destaques são os 2 primeiros usuários do array (hardcoded)
- A nova função deve ser pura (sem side effects) para facilitar testes futuros (Story 1.9)

## Dependências
- Nenhuma técnica direta, mas deve vir após 1.4 (Tailwind) para consistência visual

## Definition of Done
- Destaques refletem performance real calculada
- Membros com melhor score aparecem como destaques
- Funciona com dados reais do Supabase
