# Story 1.7 — Recovery Seguro de Senha

**Epic:** 1.0 — Tabela de Desempenho: Produção & Segurança
**Status:** Done
**Complexidade:** S (Small)
**Agente:** @dev

## Objetivo
Remover o fluxo inseguro de recuperação de senha (qualquer um reseta senha de qualquer outro) e implementar recovery seguro via Supabase Auth nativo.

## Acceptance Criteria

### Login.tsx
- [x] AC1: Modal de recuperação de senha atual removido
- [x] AC2: Link "Esqueci minha senha" abre campo de email com botão "Enviar link de recuperação"
- [x] AC3: Usa `supabase.auth.resetPasswordForEmail(email)` se email configurado no Supabase
- [x] AC4: Feedback visual: "Se o email estiver cadastrado, você receberá um link de recuperação"

### Self-Service (Usuário Logado)
- [x] AC5: Botão "Alterar Senha" acessível na interface para o usuário logado (header ou perfil)
- [x] AC6: Modal/form de troca de senha com campos: senha atual (opcional, para UX), nova senha, confirmar nova senha
- [x] AC7: Usa `supabase.auth.updateUser({ password: novaSenha })`

### AdminPanel (PARTNER)
- [x] AC8: Botão "Resetar Senha" no AdminPanel já implementado na Story 1.2 — apenas verificar que funciona corretamente
- [x] AC9: Feedback visual ao PARTNER após reset: "Senha resetada para o padrão. O membro precisará trocar no próximo login."

## Arquivos Impactados
- `components/Login.tsx` — remover modal recovery, adicionar fluxo email
- `App.tsx` ou novo componente — botão/modal "Alterar Senha" para usuário logado
- `components/AdminPanel.tsx` — verificar UX do botão reset

## Notas Técnicas
- Se o Supabase self-hosted não tiver SMTP configurado, `resetPasswordForEmail` não funcionará. Nesse caso:
  - Fallback: mostrar mensagem "Solicite ao administrador o reset da sua senha"
  - O PARTNER reseta pelo AdminPanel (já coberto na Story 1.2)
- Testar se SMTP está configurado antes de exibir opção de email

## Dependências
- Story 1.2 (Supabase Auth + Edge Function devem estar funcionando)

## Definition of Done
- Nenhum usuário consegue resetar senha de outro sem ser PARTNER
- Self-service de troca de senha funcional para usuário logado
- PARTNER consegue resetar via AdminPanel
