# Epic 2.0 — Hardening & Production-Ready

## Visão
Resolver problemas críticos de segurança, compatibilidade com Supabase Cloud e performance identificados na auditoria pós-Epic 1.0, tornando a aplicação 100% funcional em produção.

## Contexto
- **Projeto:** tabela_desempenho (React 19 + Vite + Supabase Cloud + Tailwind + Recharts)
- **Auditoria:** Pós-Epic 1.0 identificou 4 issues em 3 níveis de severidade
- **Stakeholder:** Leandro (PARTNER / Sócio Comercial)

## Issues Identificadas

### CRITICAL
1. `VITE_SUPABASE_SERVICE_ROLE_KEY` no `.env` — expõe admin key ao browser
2. `useAppState()` carrega dados antes da autenticação — gera 403 errors

### HIGH
3. `manage_user` action `create` usa INSERT direto em `auth.users` — incompatível com Supabase Cloud

### MEDIUM
4. Nenhuma memoização React (React.memo, useMemo, useCallback) — re-renders desnecessários

## Escopo

| Story | Título | Complexidade | Prioridade |
|-------|--------|-------------|------------|
| 2.1 | Segurança: service_role_key + carregamento pré-auth | S | CRITICAL |
| 2.2 | Migrar manage_user CREATE para Edge Function | M | HIGH |
| 2.3 | Performance: Memoização React | S | MEDIUM |

## Sequência de Implementação
```
2.1 → 2.2 → 2.3
```

## Critérios de Sucesso do Epic
- [x] Zero credenciais sensíveis expostas ao browser
- [x] Nenhum erro 403 no console antes/durante login
- [x] Criação de usuários via AdminPanel funcional no Supabase Cloud
- [x] Componentes memoizados para evitar re-renders desnecessários
- [x] Build sem erros + testes passando
