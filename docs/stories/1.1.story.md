# Story 1.1 — Environment Variables & Limpeza Gemini

**Epic:** 1.0 — Tabela de Desempenho: Produção & Segurança
**Status:** Ready for Review
**Complexidade:** S (Small)
**Agente:** @dev

## Objetivo
Mover credenciais do Supabase para variáveis de ambiente e remover referências não utilizadas do Gemini API.

## Acceptance Criteria

- [x] AC1: Arquivo `.env` criado com `VITE_SUPABASE_URL` e `VITE_SUPABASE_ANON_KEY`
- [x] AC2: Arquivo `.env.example` criado com placeholders (sem valores reais)
- [x] AC3: `.gitignore` inclui `.env` e `.env.local`
- [x] AC4: `supabaseClient.ts` usa `import.meta.env.VITE_SUPABASE_URL` e `import.meta.env.VITE_SUPABASE_ANON_KEY`
- [x] AC5: Bloco `define` removido de `vite.config.ts` (referências GEMINI_API_KEY)
- [x] AC6: Aplicação carrega normalmente com `npm run dev`
- [x] AC7: Build funciona sem erros com `npm run build`

## Arquivos Impactados
- `supabaseClient.ts` — substituir hardcoded por env vars
- `vite.config.ts` — remover bloco define (linhas 13-16)
- `.env` — criar
- `.env.example` — criar
- `.gitignore` — adicionar entries

## Notas Técnicas
- Vite expõe apenas variáveis com prefixo `VITE_` ao client
- O Supabase anon key é uma chave pública (segura para client), mas não deve ficar hardcoded no código-fonte para facilitar deploy em diferentes ambientes

## Dependências
- Nenhuma (primeira story)

## Definition of Done
- Aplicação funciona com credenciais via .env
- Nenhuma credencial hardcoded no código-fonte
- Build bem-sucedido
